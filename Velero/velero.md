# COMPONENTS

velero client

velero server
  BackupController
  RestoreController
  backupstoragelocationcontroller

provider plugin                       -     velero-plugin-for-aws

backupstoragelocation                 -     i think the k8s resources are backed up here

volumesnapshotlocation                -     i think the snapshots are backed up here

node-agent                            -     hosts Velero data movement controllers and launches data mover pods
                                            are used for file system backups

snapshot plugins
  velero volumesnapshotter plugin     -     this might be velero native plugin for taking snapshots
  velero csi plugin                   -     velero.io/csi-pvc-backupper - velero plugin for CSI, now merged into the main velero code 
                                           (uses volumesnapshotclass, volumesnapshot and volumesnapshotcontent)
  cloud provider plugin               -     takes a backup of the cloud providers storage (AWS EBS, GCP PD, Azure Managed Disks)

backupitemaction plugins ??

backuprepository                      -     one backup repository per namespace   THIS IS CREATED BY VELERO AS REQUIRED
  repo credentials
  repository maintenance job          -     the act of removing expired backups



secret/velero                         -     secret/velero-backups from install of objectbucketclaim/velero-backups --> ./credentials-velero
                                            --set-file on ./credentials-velero ---> secret/velero
                                            turns into the volume "cloud-credentials" and mounted to path /credentials in the velero operator
                                            the key 'cloud' becomes the file /credentials/cloud

secret/velero-repo-credentials        -     generated by velero installation, the credentials to access the repository

backups
  resources in ns
  and
  either   volume snapshot            -     native snapshot OR csi snapshot
  or       file system backup         -     aka pod volume backup

data mover
  builtin data mover                  -     which velero plugin is this ??
    uploaderType                      -     kopia (i think restic is an alternative)


storage layout
  s3://<bucket>/backups/<backup>      -     the kubernetes resources
  s3://<bucket>/kopia/<namespace>     -     the kopia repository, one per namespace
                                            contains the snapshot data if snapshot data movement was true

## INSTALL THE VELERO CLI ON YOUR MANAGEMENT SERVER
```shell
# install the velero cli
wget https://github.com/vmware-tanzu/velero/releases/download/v1.16.2/velero-v1.16.2-linux-amd64.tar.gz
tar -xvzf velero-v1.16.2-linux-amd64.tar.gz
rm -rf velero-v1.16.2-linux-amd64.tar.gz
cd velero-v1.16.2-linux-amd64/
install velero ~/.local/bin/

# configure the velero cli for CSI snapshotting
velero client config set features=EnableCSI
```

## INSTALL THE KOPIA CLI ON YOUR MANAGEMENT SERVER
```shell
wget https://github.com/kopia/kopia/releases/download/v0.21.1/kopia-0.21.1-linux-x64.tar.gz
tar -xvzf kopia-0.21.1-linux-x64.tar.gz
rm kopia-0.21.1-linux-x64.tar.gz 
cd kopia-0.21.1-linux-x64
install kopia ~/.local/bin/
```

## CREATE A credentials file
```shell
aws_access_key_id=$(kubectl get secret -n velero velero-backups -o jsonpath='{.data.AWS_ACCESS_KEY_ID}' | base64 -d)
aws_secret_access_key=$(kubectl get secret -n velero velero-backups -o jsonpath='{.data.AWS_SECRET_ACCESS_KEY}' | base64 -d)

cat <<EOF > credentials-velero
[default]
aws_access_key_id=$aws_access_key_id
aws_secret_access_key=$aws_secret_access_key
EOF
```

## OR CLI INSTALL
```shell

  -n velero \
  --provider aws \
  --plugins aplregistry.aarnet.edu.au/hub.docker.com/velero/velero-plugin-for-aws:v1.12.2 \
  --bucket velero-backups \
  --secret-file ./credentials-velero \
  --use-volume-snapshots=true \
  --features=EnableCSI \
  --use-node-agent \
  --backup-location-config region=nsw,s3ForcePathStyle="true",s3Url=http://rook-ceph-rgw-ceph-objectstore.rook-ceph:80 \
  --snapshot-location-config region=nsw \
  --image aplregistry.aarnet.edu.au/hub.docker.com/velero/velero:v1.16.2 \
  -o yaml \
  --dry-run > velero-resources-cli.yaml

#  --enable-csi-plugins # If using CSI snapshots
```

## s3cmd and kopia
```shell
namespace=infra-backups
secret=infra-backups
accesskey=$(k get secret -n $namespace $secret -o yaml | yq .data.AWS_ACCESS_KEY_ID | base64 -d)
secretkey=$(k get secret -n $namespace $secret -o yaml | yq .data.AWS_SECRET_ACCESS_KEY | base64 -d)
repo_pass=$(k get secret/velero-repo-credentials -n $namespace -o jsonpath='{.data.repository-password}' | base64 -d)
s3_endpoint=s3.opsk8stest.aarnet.net.au

cat <<EOF > ~/.s3cfg
[default]
access_key = ${accesskey}
secret_key = ${secretkey}
host_base = ${s3_endpoint}
host_bucket = ${s3_endpoint}
use_https = True
encrypt = True
gpg_command = /usr/bin/gpg
gpg_decrypt = %(gpg_command)s -d --verbose --no-use-agent --batch --yes --passphrase-fd %(passphrase_fd)s -o %(output_file)s %(input_file)s
gpg_encrypt = %(gpg_command)s -c --verbose --no-use-agent --batch --yes --passphrase-fd %(passphrase_fd)s -o %(output_file)s %(input_file)s
gpg_passphrase = ${repo_pass}
bucket_location = nsw
EOF



kopia repository connect s3 \
--endpoint=s3.opsk8stest.aarnet.net.au  \
--bucket=infra-backups \
--access-key=$accesskey \
--secret-access-key=$secretkey \
--prefix=kopia/minute-app/ \
-p ${repo_pass}

kopia snapshot list --all

# host_pods --> this one seems to be fsb backups
# snapshot-data-upload-download --> this one seems to be snapshot backups


kopia ls -l <snapshot>

kopia mount <snapshot> /tmp/mnt&

jobs

...

fg 1
ctrl-c

```
